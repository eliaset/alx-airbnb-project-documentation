\## Objective

This diagram visualizes the key interactions between different users (actors) and the Airbnb backend system. It helps to understand how users engage with major functionalities such as authentication, booking, payment, and reviews.



\## Actors

\- \*\*Guest:\*\* Can register, log in, browse properties, make bookings, payments, and leave reviews.

\- \*\*Host:\*\* Can list, update, and manage properties, and receive booking payments.

\- \*\*Admin:\*\* Monitors system activities, manages users, and moderates property listings.



\## Major Use Cases

\### User Management

\- Register / Login / Logout

\- Edit Profile

\- OAuth Authentication



\### Property Management (Host)

\- Add / Edit / Delete Property

\- View Bookings



\### Search \& Filter

\- Search for properties by location, price, and amenities



\### Booking Management

\- Book Property

\- Cancel Booking

\- View Booking Details



\### Payments

\- Process Payments

\- Check Payment Status

\- Receive Payouts



\### Reviews \& Notifications

\- Leave Review

\- View Reviews

\- Receive Notifications



\### Admin Management

\- Manage Users

\- Manage Listings

\- View Reports
